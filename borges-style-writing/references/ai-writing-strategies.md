# AI 写作策略：从故事生成器到迷宫构建器

AI（大语言模型）在长篇叙事上有致命弱点：Token 增加导致注意力发散、逻辑矛盾、人物漂移、俗套结局。

解决方案：不要强迫 AI 写"故事"，而是让它写"关于故事的描述"。

---

## 一、核心原则：Tell, Don't Show

传统创意写作教"Show, Don't Tell"（展示，不要讲述）。
博尔赫斯 + AI 语境下，反转为 **Tell, Don't Show**。

### 错误指令（导致平庸故事）

> "请写一个关于张三进入平行宇宙的故事，要有对话和动作描写。"

**结果**：张三醒来，惊讶，喝咖啡，遇到外星人……落入线性叙事陷阱，容易崩坏。

### 正确指令（偏离故事本身）

> "请撰写一篇百科全书词条，介绍'张三悖论'及其对平行宇宙伦理学的影响。"

**结果**：通过概括、引经据典、分析来侧面构建故事。静态说明性文体，AI 极难写崩，且极具博尔赫斯味。

## 二、三种"架空"视角

用 Prompt 将 AI 固定在特定的静态视角，强制偏离线性情节推进。

### 视角一：档案管理员 (The Archivist)

强制 AI 认为事情已经发生，它只是在整理旧纸堆。

**Prompt 模板**：
```
你不是小说的作者，你是一位生活在[未来年份]的档案管理员。你正在整理一份关于[过去事件]的损毁报告。

请不要描写事件的过程，只描写留下的证物：
- 一张写满乱码的收据
- 一本只有前半句话的日记
- 一段无法被播放的录音

通过对这些碎片的冷峻描述，侧面暗示当时发生的事。
```

**效果**：AI 被锁定在"事后整理"的静态视角，无法进入动态叙事。

### 视角二：伪学术评论 (Pseudo-Criticism)

博尔赫斯的拿手好戏——评论一本不存在的书。

**Prompt 模板**：
```
请写一篇关于一本名为《[虚构书名]》的小说的书评。这本书在现实中并不存在。

请详细讨论：
- 书中第[N]章关于'[核心概念]'的荒谬描写
- 引用并不存在的评论家'[虚构学者名]'的反驳观点
- 出版史上的争议（版本差异、删节、误译）

禁止描写书中的具体情节，只讨论书的结构、哲学隐喻和学术争议。
```

**效果**：AI 被锁定在"元层次"，无法进入故事内部。

### 视角三：静态切片 (Static Slice)

量子力学风格专用——不写过程，只写状态。

**Prompt 模板**：
```
请描述一个处于'[异常状态]'的[场景]。

不要写任何人走进这个场景。
只详细描写场景中的物品如何同时呈现两种矛盾状态。
用几何学和光学的术语来描写这个静止的场景。
```

**示例**：
> 描述一个处于"叠加态"的房间。物品如何同时呈现"陈旧"和"崭新"两种状态。光线如何既是明亮的又是黑暗的。

**效果**：AI 被锁定在单一时刻的静态描述，无法推进时间。

## 三、模块化拼接法 (Modular Assembly)

解决 AI 连贯性差的问题：不要让 AI 一口气写完，而是生成独立"模块"，由人类拼贴。

### 工作流程

```
┌─────────────────────────────────────────────────────────┐
│  第一步：设定公理 (World Bible)                          │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 生成 3-5 条世界观物理法则                         │   │
│  │ 例如："镜子会延迟 3 秒成像"                       │   │
│  │ 保存为 System Prompt，每次对话都包含             │   │
│  └─────────────────────────────────────────────────┘   │
│                           ↓                             │
│  第二步：生成碎片 (Artifacts)                           │
│  ┌─────────────────────────────────────────────────┐   │
│  │ Turn 1: 使用说明书警告                            │   │
│  │ Turn 2: 法医鉴定报告                              │   │
│  │ Turn 3: 经济学分析                                │   │
│  │ Turn 4: 百科词条                                  │   │
│  │ Turn 5: 学者书信                                  │   │
│  └─────────────────────────────────────────────────┘   │
│                           ↓                             │
│  第三步：人类拼贴 (Human Assembly)                      │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 将碎片按你的逻辑排列                              │   │
│  │ 添加过渡句或编者注                                │   │
│  │ 形成完整的"反小说"                                │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 碎片类型清单

| 碎片类型 | 描述 | 示例 Prompt |
|---------|------|------------|
| 使用说明书 | 某物品的官方使用指南 | "写一段关于'延迟镜子'的安全警告" |
| 法医报告 | 事件的技术性鉴定 | "写一段因镜子延迟而发生的事故的法医鉴定" |
| 经济分析 | 市场流通与价格 | "写一段这种镜子在黑市上的经济学分析" |
| 百科词条 | 中立的定义与历史 | "写一个关于'延迟镜子'的百科全书条目" |
| 学者书信 | 学术争议 | "写两位学者关于镜子原理的通信争论" |
| 目录/索引 | 某书的章节结构 | "写一本关于镜子的虚构专著的目录" |
| 讣告 | 某人的生平与死因 | "写一位研究镜子的学者的讣告" |
| 脚注 | 对某段文字的注释 | "为上述段落写三条学术脚注" |
| 勘误表 | 版本差异与错误 | "写一份该书 1923 年版的勘误表" |
| 审判记录 | 法律文书 | "写一段关于镜子事故的法庭证词摘要" |

### 示例：镜子延迟世界

**公理**：
> 本世界的镜子成像延迟 3 秒。

**碎片生成**：

1. **使用说明书**：
> "警告：请勿在镜前快速移动。本产品成像延迟约 3 秒，可能导致空间认知障碍。制造商对因延迟引发的自我误伤不承担责任。"

2. **法医报告**：
> "死者右手持剪刀刺入左胸，伤口深度 7.2 厘米。现场镜子显示死者影像于死亡后 3 秒仍保持站立姿态。鉴定结论：死者误将 3 秒前的影像认作入侵者。"

3. **经济分析**：
> "自 2019 年法规禁止延迟镜出口后，黑市价格上涨 340%。主要买家为需要'时间校准'的赌场和证券交易所。"

4. **拼贴成文**：将上述碎片以"档案集"形式排列，添加一段编者注，即成一篇博尔赫斯式短篇。

## 四、反高潮指令 (Anti-Climax Constraints)

AI 喜欢戏剧性冲突（高潮）。博尔赫斯喜欢逻辑悖论（反高潮）。

### 负面约束清单

将以下约束加入 Prompt：

```
[禁止剧情弧光]
不要遵循"开端-发展-高潮-结局"的结构。
文章应当在逻辑推演最复杂的时刻突然戛然而止，或者陷入循环。

[禁止心理描写]
不要描写人物的恐惧、爱或恨。
人物只是哲学概念的载体。

[禁止直接对话]
尽量避免使用引号进行直接对话。
如果必须有人说话，请用间接引语转述（例如："他指出……"）。

[禁止解决问题]
不要给谜题一个答案。
不要让主角"明白了什么"。
悖论应当保持为悖论。

[禁止情感升华]
不要在结尾总结意义。
不要让读者"感动"或"释然"。
保持冷漠的学术距离。
```

### 结尾策略

| 策略 | 描述 | 示例 |
|-----|------|-----|
| 戛然而止 | 在逻辑推演最复杂处停止 | "据我所知，这份手稿到此为止。" |
| 循环回到开头 | 结尾指向开头 | "而这一切，正如我在开篇提到的那本书所预言的……" |
| 元层次跳跃 | 叙述者质疑叙述本身 | "我现在怀疑，上述一切都是我自己的虚构。" |
| 悖论揭示 | 揭示无法解决的逻辑矛盾 | "这意味着，那本书的作者——是我自己。" |
| 沉默 | 以省略号或空白结束 | "关于这一点，手稿上只有一片墨渍。" |

## 五、AI 定位：平行宇宙的维基百科

核心思维转换：

| 错误定位 | 正确定位 |
|---------|---------|
| 故事生成器 | 平行宇宙的维基百科生成器 |
| 写一个故事 | 写关于这个故事的百科条目 |
| 让主角经历冒险 | 让学者讨论主角的命运 |
| 展示情节 | 引用不存在的文献 |
| 感动读者 | 困惑读者 |

### 请求生成的文本类型

优先请求这些类型，而非"故事"：

- 书评
- 脚注
- 目录
- 讣告
- 实验报告
- 法医鉴定
- 经济分析
- 百科词条
- 学者通信
- 会议记录
- 勘误表
- 索引
- 译者序言
- 出版说明

## 六、完整工作流示例

### 主题：量子观察者效应

**第一步：设定公理**

```
世界观法则：
1. 物体只有在被观察时才具有确定形态
2. 长时间无人注视的物体会逐渐"模糊化"
3. 相机拍摄的照片中，背景总是比前景更清晰（因为前景被注视）
```

**第二步：生成碎片**

Turn 1（百科词条）：
> "请写一个关于'视觉稳定剂'的百科词条——一种能让物体在无人注视时保持形态的药物。"

Turn 2（法医报告）：
> "请写一份关于某人死于'注视耗竭'的法医鉴定报告——他试图同时注视房间里的所有物品以防止它们消失。"

Turn 3（经济分析）：
> "请写一段关于'注视服务业'的经济学分析——付费雇人注视你的财产以防止贬值。"

Turn 4（学者书信）：
> "请写两位学者关于'上帝是否在观察宇宙'的通信争论。"

Turn 5（讣告）：
> "请写一位研究'模糊化'的物理学家的讣告——他最后的实验是关闭所有灯光，不再注视任何东西。"

**第三步：人类拼贴**

将上述碎片排列为"档案集"，添加编者注：

> "以下文件摘自布宜诺斯艾利斯国家档案馆'视觉稳定性研究'专题。编者对文件的真实性不作保证。"

## 七、快速参考：Prompt 结构

```
[角色设定]
你是[年份]的[职业：档案员/评论家/编辑]。

[任务类型]
请撰写一篇[文本类型：词条/报告/书评]，关于[主题]。

[架空视角]
- 档案管理员：事情已经发生，你在整理证物
- 伪学术评论：讨论一本不存在的书
- 静态切片：描述一个静止的异常状态

[负面约束]
- 禁止剧情弧光
- 禁止心理描写
- 禁止直接对话
- 禁止解决问题
- 禁止情感升华

[结尾策略]
在[逻辑最复杂处/循环回开头/元层次跳跃]结束。
```
